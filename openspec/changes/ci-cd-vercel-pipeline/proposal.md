# Proposal: CI/CD Pipeline — GitHub Actions + Vercel

## Intent

Establish a CI/CD pipeline that runs automated quality gates on every push and PR via GitHub Actions, and leverages Vercel's native GitHub Integration for all deployments — keeping deploy secrets exclusively in the Vercel Dashboard.

## Scope

### In Scope
- `.github/workflows/quality.yml` — push + PR quality gate workflow (install, test, typecheck, build)
- `.github/workflows/lighthouse.yml` — Lighthouse CI workflow triggered on `deployment_status` event
- `.lighthouserc.json` — Lighthouse performance/accessibility thresholds config at project root
- `package.json` — add `typecheck` script using `astro check`
- Branch-to-environment strategy: `main` → Production, `develop`/`staging` → Staging, PRs → Preview

### Out of Scope
- Vercel CLI in GitHub Actions (contradicts secrets-in-Dashboard preference)
- E2E tests (not yet available in stack)
- Coverage thresholds (no coverage tool configured)
- Slack/email deployment notifications
- Custom domain setup or DNS changes

## Capabilities

### New Capabilities
- `ci-quality-pipeline`: GitHub Actions workflow for quality gates (tests, typecheck, build) on every push and PR
- `ci-lighthouse-pipeline`: Lighthouse CI workflow triggered on Vercel deployment status, auditing real preview/production URLs
- `vercel-branch-strategy`: Branch-to-environment mapping for Vercel (main → Production, develop/staging → Staging, PRs → Preview)

### Modified Capabilities
- `package.json` — adds `typecheck` script (`astro check`); no behavioral change to existing scripts

## Approach

Vercel Native GitHub Integration handles all deployments automatically. GitHub Actions runs quality gates independently in parallel. Lighthouse CI fires after Vercel reports a successful deployment via `deployment_status` webhook, auditing the real deployed URL — no deploy tokens required in GitHub Secrets.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `.github/workflows/quality.yml` | New | Quality gate: pnpm install → vitest → astro check → astro build |
| `.github/workflows/lighthouse.yml` | New | Lighthouse CI on `deployment_status` using `treosh/lighthouse-ci-action` |
| `.lighthouserc.json` | New | Defines performance, accessibility, and best-practice score thresholds |
| `package.json` | Minor | Adds `typecheck` script (`astro check`) |
| Vercel Dashboard | Config | Must have GitHub Integration enabled; branch → environment mapping set |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Lighthouse never fires if Vercel GitHub Integration is disabled | Medium | Document as prerequisite; add check to proposal success criteria |
| `astro check` false-positives on virtual modules if replaced with bare `tsc` | Low | Use `astro check` exclusively, not `tsc --noEmit` directly |
| Quality and deploy run in parallel — deploy not blocked by failing tests | Medium | Accepted tradeoff; no deploy secrets in GH is a hard constraint |
| Undefined Lighthouse thresholds → workflow never fails | Low | Define explicit thresholds in `.lighthouserc.json` before merge |

## Rollback Plan

All changes are additive (new files + one `package.json` script). Rollback is a single PR that removes `.github/workflows/quality.yml`, `.github/workflows/lighthouse.yml`, `.lighthouserc.json`, and reverts the `typecheck` script from `package.json`. No database migrations or infrastructure teardown required.

## Dependencies

- Vercel GitHub Integration enabled in the Vercel Dashboard (required for `deployment_status` events)
- `@astrojs/check` available via `astro check` CLI (included with Astro 7)
- `treosh/lighthouse-ci-action` — GitHub Action, no install needed
- Node >= 22.12.0 pinned in workflows to match engine constraint
- pnpm version pinned to match current `packageManager` field in `package.json`

## Success Criteria

- [ ] `quality.yml` runs on every push to `main`, `develop`, `staging`, and on all PRs
- [ ] All three quality gates pass: `pnpm test`, `astro check`, `pnpm build`
- [ ] `lighthouse.yml` fires automatically after a Vercel preview deployment completes
- [ ] Lighthouse scores meet defined thresholds in `.lighthouserc.json` (performance ≥ 90, accessibility ≥ 90)
- [ ] No deploy secrets or Vercel tokens stored in GitHub Secrets
- [ ] PR preview URLs continue to be generated by Vercel natively (unchanged behavior)
- [ ] `pnpm typecheck` runs successfully via `astro check` locally and in CI
